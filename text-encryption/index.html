How It Works

Encryption Process

  1. Generate random 256-bit AES key
  2. Generate random 96-bit nonce
  3. Encrypt text with AES-GCM
  4. Encrypt AES key with RSA public key
  5. Share: ciphertext + nonce + encrypted key

Why Hybrid?

AES: Fast, secure for large data

RSA: Secure key exchange

Combined: Best of both worlds

Security Features

  • AES-GCM: Encryption + Authentication
  • Random Nonce: Each encryption is unique
  • OAEP Padding: RSA secure padding
  • SHA-256: Hash algorithm

Real-World Use

This approach is used in:

  • HTTPS/TLS
  • Email encryption (PGP)
  • Cloud storage
  • Secure messaging
